# MemPalace Bot 🤖

Telegram-бот, который общается с искусственным интеллектом локальным или через API, ищет информацию в ваших личных заметках (MemPalace), скачивает видео с YouTube, делает транскрипции и многое другое. Работает только у вас на компьютере. Пока компьютер работает, работает и бот. Как сделать чтоб работал, даже с закрытой крышкой? Установите Amphitamin и настройте на это.

## 🔧 Что нужно для установки

- **Компьютер Mac** (работает на Intel (медленнее в 4 раза) и Apple Silicon M1/M2/M3)
- **Доступ в интернет**
- **Аккаунт Telegram**
- **Умение копировать и вставлять текст** — всё остальное сделают инструкции ниже

## 📦 Пошаговая установка

### Шаг 1. Установите Homebrew (если ещё нет)

Откройте программу **Терминал** (нажмите `⌘ + Пробел`, введите `Терминал` и нажмите Enter).

Скопируйте и вставьте эту команду в Терминал, нажмите Enter:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Если спросит пароль — введите пароль от вашего Mac (символы не будут видны, это нормально). Процесс может занять несколько минут.

### Шаг 2. Установите Python и FFmpeg

Вставьте в Терминал:

```bash
brew install python@3.12 ffmpeg
```

### Шаг 3. Установите MemPalace

```bash
brew install anomalyco/tap/mempalace
```

### Шаг 4. Скачайте бота

Откройте папку, в которую хотите установить бота. Например:

```bash
cd ~/Documents
git clone https://github.com/Eugen-I/mempalace_bot.git
cd mempalace_bot
```

### Шаг 5. Создайте виртуальное окружение

```bash
python3.12 -m venv venv
source venv/bin/activate
```

### Шаг 6. Установите зависимости

```bash
pip install -r requirements.txt
```

### Шаг 7. Создайте файл с ключами

Скопируйте образец:

```bash
cp .env.example .env
```

Откройте файл `.env` в любом текстовом редакторе и заполните:

```
TELEGRAM_BOT_TOKEN= — ваш токен от @BotFather (как получить — ниже)
ADMIN_ID= — ваш ID в Telegram (как узнать — ниже)
ALLOWED_USERS= — ID других пользователей (через запятую)
GEMINI_API_KEY= — ключ от Google Gemini (как получить — ниже)
OPENAI_API_KEY= — ключ от OpenAI (если используете GPT-4o; получить на https://platform.openai.com/api-keys)
```

> 💡 Если ваша папка MemPalace находится **не** в `~/Documents/mempalace`, добавьте в `.env` строку:
> ```
> MEMPALACE_DATA_DIR=/путь/к/вашей/папке/mempalace
> ```

**Как получить TELEGRAM_BOT_TOKEN:**
1. Откройте Telegram, найдите @BotFather
2. Напишите `/newbot`, придумайте имя (например, MyMemBot)
3. BotFather пришлёт токен — скопируйте его (выглядит как `123456:ABCdef...`)
4. Вставьте вместо `your_telegram_bot_token`

**Как узнать свой ADMIN_ID:**
1. Откройте Telegram, найдите @userinfobot
2. Нажмите /start — бот покажет ваш ID (число, например 123456789)
3. Вставьте это число вместо 0 в строку ADMIN_ID

**Как получить GEMINI_API_KEY:**
1. Откройте site: https://aistudio.google.com/apikey
2. Нажмите «Create API key»
3. Скопируйте ключ и вставьте в .env

### Шаг 8. Запустите бота

```bash
python3 main.py
```

Если всё сделано правильно, вы увидите:
```
🦾 MemPalace запущен.
```

Теперь откройте Telegram, найдите своего бота и нажмите /start.

## 🎮 Как пользоваться

После запуска бот покажет кнопки быстрого доступа:

| Кнопка | Что делает |
|--------|-----------|
| 🆕 Новый диалог | Создаёт новый чат с ИИ |
| 📂 Список чатов | Показывает ваши диалоги |
| ⚙️ Настройки | Выбор модели ИИ, голосовой режим |
| 🔍 Поиск по крылу | Ищет в ваших заметках по теме |
| 🔄 Синхронизация | Сохраняет диалог в MemPalace |
| 📹 Скачать видео | Скачивает YouTube-ролик (480/720p) |
| 🎵 Скачать MP3 | Скачивает аудио с YouTube и предлагает транскрипцию |

### 🤖 Общение с ИИ

Просто пишите сообщения — бот отвечает через ИИ (Gemini / Ollama / OpenAI).

**Специальные команды в сообщениях:**

| Символ | Действие |
|--------|----------|
| `!` текст | Быстро сохранить заметку в `my_notes` |
| `!!` | Извлечь инсайт из последнего ответа ИИ и сохранить в папку `insights` |
| `???` | Сохранить последний ответ как исследование в `research` |
| `/search текст` | Поиск по базе MemPalace |
| `/search --wing dreams текст` | Поиск в конкретном крыле (dreams / projects / philosophy / creative / psychology) |

Также можно указать крыло через `/`: например, `/philosophy: что такое бытие?`

### 📄 Работа с PDF

Просто отправьте боту PDF-файл — он:
1. Извлекает текст (с OCR на немецком, русском и английском)
2. Определяет тип документа (медицина / финансы / личное)
3. Делает точный анализ с сохранением всех чисел, дат, терминов
4. Сохраняет оригинал в архив

**Команды:**
- `/pdfs` — список сохранённых PDF, можно переанализировать любой
- Кнопка «Сравнить» после анализа — задать вопрос по документу с таймером 2 минуты
- Автоматическая обработка больших PDF через чанкинг

### 🎤 Голосовые сообщения

Отправьте голосовое сообщение — бот:
1. Распознаёт речь через Whisper (локально, на вашем Mac)
2. Отвечает голосом или текстом (настраивается в ⚙️ Настройки)

Режимы голоса: только текст, только голос, и то и другое.

### 📸 Анализ фото

Отправьте фото — бот проанализирует его, если в тексте есть слова «фото», «анализируй», «image» и т.п.
Поддерживает все модели с multimodальными способностями.

### 🧠 Долговременная память

Бот автоматически:
1. Извлекает ключевые факты из каждого диалога
2. Сохраняет их в отдельную базу памяти
3. При следующих вопросах находит релевантные факты и подставляет их в контекст ИИ

Это работает вместе с MemPalace: каждые 5 сообщений диалог автоматически синхронизируется в базу.

### 🔄 Реакции на сообщения

После ответа ИИ поставьте реакцию:
- 👍 — сохранить в `my_notes`
- ❤️ — сохранить как инсайт в `insights`
- 🤷 — сохранить как исследование в `research`

### 💻 Интерфейс командной строки (CLI)

Бота можно использовать и без Telegram — через терминал:

```bash
python3 cli_ask.py
```

Поддерживает те же функции: ИИ, поиск по MemPalace, заметки, озвучка.

**Команды управления дворцом (CLI):**
| Команда | Действие |
|---------|----------|
| `/palace` | Показать список команд дворца |
| `/status` | Статистика MemPalace (крылья, комнаты, количество записей) |
| `/wings` | Список всех крыльев |
| `/rooms dreams` | Комнаты в указанном крыле |
| `/taxonomy` | Полная иерархия дворца |
| `/traverse комната` | Обход графа знаний из комнаты |
| `/tunnels dreams philosophy` | Поиск связей между двумя крыльями |
| `/wakeup` | Загрузить дворец в активный контекст |
| `/repair` | Перестроить векторный индекс |
| `/compress` | Сжать хранилище |

### 🏰 Дворец в Telegram-боте

Кнопка `🏰 Дворец` (или команда `/palace`) открывает меню:
- **Статус** — статистика дворца
- **Крылья** — список всех крыльев
- **Полная иерархия** — таксономия
- **Обслуживание** — repair, compress, wake-up
- **Инструкции** — справка по MemPalace

## 📂 Папка `_scripts` — готовые скрипты

Все скрипты работают по двойному клику (`.command`). Ничего вводить в терминале не нужно — просто нажмите на файл в Finder.

### 🚀 Запуск и управление

| Скрипт | Когда запускать | Что делает |
|--------|----------------|------------|
| **Ollama_Start** | Перед первым запуском бота | Запускает Ollama (сервер для локальных ИИ-моделей) |
| **start_bot** | Каждый раз для запуска бота | Запускает Telegram-бота |
| **PalaceManager_TelegrammBot** | Если нужен полный контроль | Меню: запуск (интерактивно / в фоне), остановка, логи, очистка |
| **KillPalaceBot** | Если бот завис и не отвечает | Принудительно завершает все процессы бота |
| **Switch_AI** | Чтобы сменить модель ИИ | Выбор активной модели из установленных в Ollama / OpenAI / Gemini |
| **setup_env** | При первой настройке | Меню для заполнения `.env` (токены, ключи, ID) |
| **RestoreConfig** | Если пропали алиасы в терминале | Восстанавливает команду `mempalace` и другие настройки |

### 🧠 Управление памятью MemPalace

| Скрипт | Когда запускать | Что делает |
|--------|----------------|------------|
| **update_memory** | После добавления заметок | Обновляет базу MemPalace из папки `my_notes` |
| **update_memory_insights_research** | После инсайтов и исследований | Обновляет базу из папок `insights` и `research` |
| **update_memory_wingMode** | Для разметки по темам | Индексирует заметки с авто-разделением по крыльям |
| **check_memory** | Чтобы увидеть состав базы | Показывает статус MemPalace и список файлов |
| **clear_memory** | Чтобы сбросить базу | Полностью очищает память MemPalace (файлы остаются) |
| **Ask_Memory** | Для быстрого диалога с ИИ | Запускает CLI-режим (общение без Telegram) |
| **Run_Extraction** | Для извлечения мыслей | Запускает скрипт экстракции важных фактов |

### 🔍 Поиск и инструменты

| Скрипт | Когда запускать | Что делает |
|--------|----------------|------------|
| **find_memory** | Когда нужно найти в заметках | Бесконечный поиск с настройкой количества и порога |
| **go_to_memory** | Чтобы открыть терминал в базе | Переходит в папку MemPalace с активированным окружением |
| **Archive_Chat** | Чтобы сохранить диалог в заметки | Переносит выбранный чат из бота в `my_notes` |
| **Daily_Note** | Для ежедневной записи | Открывает daily-заметку (если настроена в MemPalace) |

### ⚙️ Модели и обслуживание

| Скрипт | Когда запускать | Что делает |
|--------|----------------|------------|
| **Manage_Wechsel_Models** | Чтобы добавить/удалить модель | Меню управления списком моделей в `models.json` |
| **Ollama_Cleaner** | Когда мало места на диске | Показывает установленные Ollama-модели и удаляет ненужные |
| **import_notes** | Если переносите заметки из Apple Notes | Импортирует заметки из Apple Notes в MemPalace |

## 🖥️ Приложение MemPalace Bot.app

В корне проекта находится **MemPalace Bot.app** — нативная программа для macOS.
Все 22 скрипта из папки `_scripts` собраны в одном окне с тёмной темой.

### Как пользоваться

1. Перетащите `MemPalace Bot.app` в любую удобную папку (Рабочий стол, Программы и т.д.)
2. Дважды кликните — откроется окно с 4 колонками:
   - **🚀 Запуск** — старт бота, Ollama, управление
   - **🧠 Память** — обновление и проверка MemPalace
   - **🔍 Инструменты** — поиск, архивация, daily note
   - **⚙️ Обслуживание** — модели, очистка, настройка
3. Кликните на любой скрипт — приложение само определит режим:
   - 🪟 **Терминал** — для интерактивных скриптов (выбор модели, настройка)
   - ⚡ **Фон** — вывод прямо в окно приложения (обновление памяти, запуск бота)
4. ⏹ **Стоп** — завершить все фоновые задачи
5. 📎 **.command** — добавить свой скрипт из любой папки
6. 📝 **Вставить код** — создать скрипт прямо из приложения (название + bash-код + режим)

### Где хранятся данные

- Пользовательские скрипты: `~/Library/Application Support/MemPalaceBot/`
- Все основные скрипты встроены в приложение — не теряются при обновлении

### Сборка (для разработчиков)

```bash
osacompile -o MemPalace Bot.app -e 'on run' -e 'do shell script "python3 " & quoted form of (POSIX path of (path to me) & "Contents/Resources/Scripts/mempalace_gui.py") & " &> /dev/null &"' -e 'end run'
cp _scripts/mempalace_gui.py "MemPalace Bot.app/Contents/Resources/Scripts/"
cp _scripts/*.command "MemPalace Bot.app/Contents/Resources/Scripts/"
```

## 🧹 Если что-то пошло не так

**Бот не отвечает:**
- Проверьте, запущен ли скрипт (в Терминале должно быть видно `Bot polling started`)
- Проверьте, правильно ли введён токен в `.env`
- Проверьте, что бот запущен только один раз — два экземпляра конфликтуют
- Используйте `KillPalaceBot.command` для принудительной остановки, затем запустите снова

**Ошибка «Module not found»:**
- Выполните `pip install -r requirements.txt` ещё раз
- Убедитесь, что виртуальное окружение активировано: `source venv/bin/activate`

**Ошибка «venv/bin/activate: No such file or directory»:**
- Виртуальное окружение не создано. Выполните `python3.12 -m venv venv` в папке проекта

**Ошибка «TELEGRAM_BOT_TOKEN not found»:**
- Проверьте, что файл `.env` существует в папке `~/Documents/mempalace/`
- Проверьте, что в нём есть строка `TELEGRAM_BOT_TOKEN=ваш_токен`

**Бот запущен, но не видит команды:**
- Убедитесь, что вы пишете боту в Telegram, а не ищете его в поиске
- Нажмите `/start` для инициализации

**PDF не обрабатывается:**
- Убедитесь, что установлен Tesseract: `brew install tesseract tesseract-lang`
- Большие PDF (50+ страниц) обрабатываются дольше — подождите

**YouTube не скачивается:**
- Убедитесь, что установлен `ffmpeg`: `brew install ffmpeg`
- Ссылка должна быть на видео, не на плейлист

**Ошибка «No module named 'yt_dlp'»:**
- Выполните `pip install -r requirements.txt` — все зависимости установятся заново

**Как обновить бота:**
- Выполните в Терминале: `cd ~/Documents/mempalace_bot && git pull && source venv/bin/activate && pip install -r requirements.txt`

**Как найти логи:**
- Логи бота: файл `bot_debug.log` в папке проекта
- Логи запуска: файл `nohup.out` в папке проекта (удаляется автоматически)
